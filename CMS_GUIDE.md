# Studio 内容后台使用说明

访问地址：`/studio`

## 1. 内容放在哪里

后台新增的内容不会写进代码目录，而是保存在服务器：

```text
data/content-store.json
```

后台上传的图片会保存在：

```text
data/uploads/
```

这两个目录已经属于运行数据。以后更新代码、重新部署时，不要删除 `data/`。

## 2. 如何区分 Journal、Notes、Garden

在 `/studio` 的“内容”页可以选择“栏目”：

- `Journal 长文章`：正式文章、长篇随笔、技术记录，发布后出现在 `/journal` 和首页。
- `Notes 短随笔`：短想法、灵感、摘录，发布后出现在 `/notes` 和首页。
- `Garden 半隐藏`：不进入首页公开流，只出现在底部小句号入口 `/garden`。

如果栏目选择 `Garden`，系统会自动把可见性设为“半隐藏花园”。

## 3. 图片如何区分是哪一张

每篇内容只有一个“封面图片 URL”，它决定：

- 首页图片卡片显示哪张图；
- 列表页缩略图显示哪张图；
- 文章详情页头图显示哪张图。

操作方式：

1. 点击“上传封面”。
2. 选择本地图片。
3. 上传成功后，后台会自动把地址填入“封面图片 URL”。
4. 保存草稿或发布。

正文里的图片则用 Markdown 写：

```md
![图片说明](/api/uploads/xxxx.jpg)
```

封面图控制入口视觉，正文图片控制文章阅读区域。

## 4. 草稿和发布

- `保存草稿`：只在后台可见，前台不会显示。
- `发布`：保存并立即出现在对应前台页面。
- `删除`：从后台内容库删除，前台也会消失。

建议写作流程：

1. 新文章 / 新随笔 / 新花园。
2. 上传封面。
3. 写标题、摘要、标签、正文。
4. 保存草稿。
5. 预览字段确认无误后发布。

## 5. Markdown 正文示例

```md
## 今天的记录

这是一段正文，支持中文、英文和链接。

> 这是一段引用。

![一张图片](/api/uploads/example.jpg)

```ts
console.log("hello room")
```
```

## 6. 评论和聊天室管理

`/studio` 顶部有三个页签：

- `内容`：新增、编辑、发布文章。
- `评论`：将评论设为 visible、hidden 或 deleted。
- `聊天室`：将聊天消息设为 visible、hidden 或 deleted。

`hidden` 适合临时隐藏，`deleted` 适合删除展示。
